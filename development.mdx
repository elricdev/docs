</Info>

## Self-Hosting Setup

Follow these steps to run Elric on your own server:

**Step 1**: Clone the repository:

<CodeGroup>

```bash git
git clone https://github.com/elricbot/elric-discord-bot.git
cd elric-discord-bot
```

</CodeGroup>

**Step 2**: Install dependencies:

<CodeGroup>

```bash npm
npm install
```

```bash yarn
yarn install
```

</CodeGroup>

**Step 3**: Configure environment variables:

Create a `.env` file with your Discord bot token and other required settings:

```env
DISCORD_TOKEN=your_bot_token_here
DATABASE_URL=file:./database.db
LAVALINK_HOST=localhost
LAVALINK_PORT=2333
LAVALINK_PASSWORD=youshallnotpass
```

**Step 4**: Set up the database:

```bash
npm run prisma:generate
npm run prisma:migrate
```

**Step 5**: Start the bot:

```bash
npm run start
```

## Development Environment

For contributing to Elric's development:

```bash
npm run dev
```

This runs the bot in development mode with hot reloading and debug logging.

### Database Management

Elric uses Prisma with SQLite for data management:

```bash
# View database in Prisma Studio
npm run prisma:studio

# Reset database
npm run prisma:reset

# Generate Prisma client
npm run prisma:generate
```

## Configuration Files

Key configuration files you may need to modify:

- `config/bot.config.js` - Bot settings and feature toggles
- `config/commands.config.js` - Command permissions and cooldowns
- `prisma/schema.prisma` - Database schema
- `.env` - Environment variables

## Troubleshooting

<AccordionGroup>
  <Accordion title="Bot not responding to commands">
    Check that:
    - Bot has proper permissions in your server
    - Bot token is valid in .env file
    - Database connection is working
  </Accordion>

  <Accordion title="Music commands not working">
    Ensure Lavalink server is running and properly configured:
    ```bash
    java -jar Lavalink.jar
    ```
  </Accordion>

  <Accordion title="Database errors">
    Try resetting the database:
    ```bash
    npm run prisma:reset
    npm run prisma:migrate
    ```
  </Accordion>
</AccordionGroup>
